# Bug Report: GET /api/song/{song_id}/info with an expired Bearer token returns 200 OK (Should return 401 Unauthorized)

## Summary
The endpoint `/api/song/{song_id}/info` incorrectly returns **200 OK** when called with an **expired Bearer token**.  
The expected behavior is to return **401 Unauthorized** and prevent access to song information.

## Steps to Reproduce
1. Execute a **GET** request to `/api/song/{song_id}/info`
2. Use a **valid song ID**
3. Provide an **expired Bearer token** in the Authorization header
4. Submit the request

## Expected Result
- Response status: **401 Unauthorized**  
- No song information should be displayed  

## Actual Result
- Response status: **200 OK**  
- HTML content is returned instead of an authorization error  
- The API fails to enforce token expiration validation
