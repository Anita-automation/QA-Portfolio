# Bug Report: GET /api/song/{song_id}/info with an expired Bearer token returns 200 OK (Should be 401 Unauthorized)

## Summary
The API incorrectly returns **200 OK** when requesting song information using an **expired Bearer token**.  
A **401 Unauthorized** response is expected for requests made with invalid or expired authentication tokens.

## Environment
- QA Environment: Koel  
- Endpoint: `GET /api/song/{song_id}/info`  
- Authentication: Expired Bearer token  

## Preconditions
- Valid user account exists  
- Valid song ID exists in the system  
- Postman environment configured  
- Endpoint is available  
- Expired Bearer token is used  

## Steps to Reproduce
1. Send a **GET** request to `/api/song/{song_id}/info` using a valid song ID  
2. Include an **expired Bearer token** in the authorization header  
3. Submit the request  
4. Observe the response  

## Expected Result
- Response status: **401 Unauthorized**
- Response body indicates that authentication is required 
- No song information is displayed  

## Actual Result
- Response status: **200 OK** 
- This behavior violates expected authentication and security requirements
